# CLAUDE.md — BKR FFT Teslja

Цей файл пояснює Claude Code структуру і правила роботи з репозиторієм бакалаврської роботи.

## Тема і контекст

**Тема:** Візуальне моделювання роботи обчислювача ШПФ під керуванням потоку даних  
**Спеціальність:** 122 Комп'ютерні науки, НУ «Львівська політехніка»  
**Мова звіту:** Українська (науковий стиль, безособова форма)  
**Формат:** LaTeX, компілятор pdfLaTeX або LuaLaTeX

## Зв'язок з симулятором (../симулятор або батьківська папка)

Реалізований Next.js симулятор (`DataflowRuntime`, `scheduler.ts`, `generateDFT4x4.ts`) є
**програмною реалізацією** що описується в Розділі 3 цієї БКР. При написанні тексту:
- Розділ 3 описує архітектуру симулятора як «програмну реалізацію»
- `NodeKind` (source, sink, butterfly, dft4, twiddle) → відповідають блокам у розділі 2
- `DataflowRuntime.tick()` → описується в підрозділі 3.5 як «потокова модель виконання»
- `generateDFT4x4.ts` → це і є реалізація алгоритму з розділу 2

## Правила написання тексту

### Мова і стиль
- Мова: **українська**
- Форма: **безособова** («розглядається», «визначається», «обчислюється»)
- НЕ використовувати: «я», «ми», «автор вважає»
- НЕ використовувати: розмовні звороти, жаргон
- Скорочення: розшифровувати при першому вживанні

### LaTeX конвенції
- Рисунки: `\begin{figure}[H]` з підписом `Рис.~\ref{fig:...} — Назва`
- Таблиці: `\begin{table}[H]` з підписом `Таблиця~\ref{tab:...}`
- Формули: нумеруються в межах розділу `(2.1)`, `(2.2)`, ...
- Посилання: `\cite{key}` у квадратних дужках через `[1]`, `[2]`
- Після кожного розділу — підрозділ `\subsection*{Висновки до розділу N}`

### Вимоги до оформлення (кафедра АСУ)
- Шрифт: Times New Roman 14pt
- Інтервал: 1.5
- Поля: лівое 25мм, праве 10мм, верхнє/нижнє 20мм
- Загальний обсяг: 70-90 стор. без додатків
- Джерел: не менше 20

## Технічні деталі симулятора (для точного опису в тексті)

```
DataflowRuntime.tick(dt, maxFires):
  - Доставляє токени з FIFO-черг з урахуванням edge.delay
  - Запускає вузли коли всі входи готові та вузол не блокований latency
  - NodeKind: BUTTERFLY (DIF метелик), DFT4 (оптимізований 4-точковий),
    TWIDDLE (множення на e^{-j2πk/N})
  - Емітує події: onToken, onFire, onOutput

generateDFT4x4:
  - 4 рядкових DFT4 → twiddle фактори W16^(n1*k2) → 4 стовпцевих DFT4
  - edge.delay = 600, latency DFT4 = 400, latency twiddle = 200

Метрики (simStore):
  - EMA throughput/latency/queue size
  - Node activity heatmap (decay 0.92 per frame)
  - compareWithFFT() з epsilon 1e-9
```
